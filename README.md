# API Security Risk Analysis

This project presents an API Security Risk Analysis conducted on a public test API using safe and ethical testing methods.

## API Tested
https://jsonplaceholder.typicode.com

## Tools Used
- Postman  
- Browser Developer Tools  

## Key Findings
- No Authentication Required (High Risk)  
- Sensitive Data Exposure (Medium Risk)  
- No Rate Limiting (Medium Risk)  
- Improper Error Handling (Low Risk)  

## Objective
The objective of this task was to analyze API security risks, evaluate access control, and suggest practical remediation steps without performing any harmful actions.

## Disclaimer
This analysis was conducted using read-only and ethical testing methods on a public test API. No exploitation or harmful activity was performed.
